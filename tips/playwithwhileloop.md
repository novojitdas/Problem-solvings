### While Loop 

#### 1. while loop ও Increment and Decrement Operator কোডে ব্যবহার করার ট্রিকস 
*** 
**Prerequisites:** (While Loop,Increment and Decrement Operator,Pre and Post Usage)  

##### Sample Code:
```cpp
    #include <iostream>
    using namespace std;

    int main()
    {
        int precount = 0;
        int postcount = 0;
        int pre, post;
        cin >> pre >> post;
        while (--pre) // if n=1 and need to go till n-1 th
        {
            precount++;
        }
        while (post--) // if need n th
        {
            postcount++;
        }

        cout << "precount: " << precount << endl;
        cout << "postcount: " << postcount << endl;

        return 0;
    }
```
    OUTPUT:
    4
    4
    precount: 3
    postcount: 4


**এই আর্টিকেলের মেইন উদ্দেশ্যঃ** pre and post নিয়ে আমরা কিভাবে আমাদের টেস্ট কেস/ লিমিটেড কেসে কাজে লাগাতে পারি সেটা সম্পর্কে ধারনা নেওয়া। আমি এখানে শুধু decrement নিয়ে কাজ করেছি। incrementও অনেকটা সেইম।

উপরের স্যাম্পল কোডের মেইন ফাংশনের ভিতরে আমরা দুইটি লোকাল ভেরিয়েবলঃ **precount,postcount** দেখতে পাচ্ছি।

এখানে, 
**১) precount**: while লুপের ভিতরের pre decrement কতবার হলো তা হিসাব করবে। 

**২) postcount**: while লুপের ভিতরের post decrement কতবার হলো তা হিসাব করবে।

এখন আমি আমার বুঝার সুবিধার্থে উভয় ভেরিয়েবলে ইনপুট হিসেবে 4 নিলাম। তোমরা কমিয়ে বাড়িয়ে দেখতে পারো। 

এখন এই কোডের উল্লেখযোগ্য অংশ হচ্ছে **while loop** কিভাবে কাজ করে। 

প্রথম, **while (--precount)** 

    1   precount = precount - 1;
    2   while(precount)
    3

এখানে, **while (--precount)**  এর কাজ হচ্ছে। প্রথমে প্রি ডিক্রিমেন্ট অপারেটর ইউজ করার কারনে প্রথমে ভেরিয়েবল থেকে ১ কমবে তারপর while কন্ডিশন চেক করবে।

২য়, **while (postcount--)**

    1   while(postcount)
    2   postcount = postcount - 1;
    3

এখানে প্রথমেই পোস্ট কাউন্ট ভেরিয়েবল কন্ডিশন চেক করবে। তারপর, পোস্ট কাউন্ট থেকে ১ কমিয়ে দিবে। 

মজার ব্যপার হলো এখানে যদি আমরা আউটপুট দেখার জন্য কোন প্রিন্ট ভেরিয়েবল ইউজ করি তাহলে। পোস্টকাউন্ট ভেরিয়েবলটি শুন্য শো করলেও, প্রিকাউন্ট ভেরিয়েবল শুন্য শো করবে না। কারন হচ্ছে, এটি while loop এর কন্ডিশন আগে চেক করার আগে ১ কমায়। লাস্টে যখন আমাদের ১ থাকবে তখন এটি কমিয়ে শুন্য হয়ে যাবে। আমরা জানি যে, while(0) = false, অর্থাৎ কোন কন্ডিশন রান করবে না। সেই কারনে লুপের ভিতরে থাকা কোড ব্লকটি রান করবে না। তাই কোন আউটপুট আমরা দেখতে পাই না। 

    precount = 1-1; // precount = 0
    while(0); 
    {
        কোড ব্লক রান করবে না 
    }

অন্যদিকে পোস্টকাউন্টে আগে কন্ডিশন চেক করে, তারপর ভেরিয়েবল থেকে ১ কমে শুন্য হয়। আগে কন্ডিশন চেক করা অবস্থায় এটি ১ থাকার কারনে। লুপের ব্লকের ভিতরের কার্যক্রম সম্পন্ন হয়। কিন্তু প্রিকাউন্টে সেটা হয় না। 

    while(1)
    postcount = 1-1; // postcount = 0
        
        {
            কোড ব্লক রান করবে 
        }

এই কারনে পোস্টকাউন্টে আমরা ইনপুটের সমান কাউন্ট নাম্বার পেলেও প্রি কাউন্টে একটা কম পাই (n-1)। তাই পোস্ট ডিক্রিমেন্ট অপারেটর ইউজ করাই Better। 

এখন কথা হচ্ছে, **কখন এটা ইউজ করব?** 

সহজ উত্তর, যখন তোমার দরকার হবে। যদিও এটা সফটওয়্যার ইঞ্জিনিয়ারিংয়ের "**KISS**" (Keep It Simple) প্রিন্সিপালের মধ্যে পরে না। কারন অধিকাংশ ডেভেলপারদের while বুঝতে কিছুটা কষ্ট হয়। তবে কম্পিটেটিভ প্রোগ্রামিং অথবা প্রবলেম সলভিং এ এই টেকনিক ইউজ করা যেতে পারে।
